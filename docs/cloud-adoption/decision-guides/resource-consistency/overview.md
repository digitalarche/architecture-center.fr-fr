## <a name="basic-grouping"></a><span data-ttu-id="adf31-116">Regroupement de base</span><span class="sxs-lookup"><span data-stu-id="adf31-116">Basic grouping</span></span>

<span data-ttu-id="adf31-117">Dans Azure, les [groupes de ressources](/azure/azure-resource-manager/resource-group-overview#resource-groups) sont un mécanisme d’organisation des ressources de base permettant de regrouper logiquement les ressources dans un abonnement.</span><span class="sxs-lookup"><span data-stu-id="adf31-117">In Azure, [resource groups](/azure/azure-resource-manager/resource-group-overview#resource-groups) are a core resource organization mechanism to logically group resources within a subscription.</span></span>

<span data-ttu-id="adf31-118">Les groupes de ressources servent de conteneurs pour les ressources ayant un cycle de vie commun ou des contraintes de gestion partagées telles que les exigences en matière de stratégie ou de contrôle d’accès en fonction du rôle (RBAC).</span><span class="sxs-lookup"><span data-stu-id="adf31-118">Resource groups act as containers for resources with a common lifecycle or shared management constraints such as policy or role-based access control (RBAC) requirements.</span></span> <span data-ttu-id="adf31-119">Les groupes de ressources ne peuvent pas être imbriqués et les ressources ne peuvent appartenir qu’à un seul groupe de ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-119">Resource groups can't be nested, and resources can only belong to one resource group.</span></span> <span data-ttu-id="adf31-120">Certaines actions peuvent agir sur toutes les ressources contenues dans un groupe de ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-120">Some actions can act on all resources in a resource group.</span></span> <span data-ttu-id="adf31-121">Par exemple, la suppression d’un groupe de ressources supprime toutes les ressources dans ce groupe.</span><span class="sxs-lookup"><span data-stu-id="adf31-121">For example, deleting a resource group removes all resources within that group.</span></span> <span data-ttu-id="adf31-122">Il existe des modèles communs lors de la création de groupes de ressources, généralement répartis en deux catégories :</span><span class="sxs-lookup"><span data-stu-id="adf31-122">There are common patterns when creating resource groups, commonly divided into two categories:</span></span>

- <span data-ttu-id="adf31-125">Charges de travail informatiques de type agile : Elles se concentrent sur les applications cloud pour les clients externes.</span><span class="sxs-lookup"><span data-stu-id="adf31-125">Agile IT workloads: Focus on external customer-facing cloud applications.</span></span> <span data-ttu-id="adf31-126">Ces groupes de ressources reflètent souvent les couches fonctionnelles de déploiement (comme la couche web ou la couche application) et de gestion.</span><span class="sxs-lookup"><span data-stu-id="adf31-126">These resource groups often reflect the functional layers of deployment (such as web tier or app tier) and management.</span></span>

## <a name="deployment-consistency"></a><span data-ttu-id="adf31-127">Cohérence du déploiement</span><span class="sxs-lookup"><span data-stu-id="adf31-127">Deployment consistency</span></span>

<span data-ttu-id="adf31-128">S’appuyant sur le mécanisme de regroupement des ressources de base, la plupart des plateformes cloud offrent un système permettant d’utiliser des modèles de déploiement de vos ressources dans l’environnement cloud.</span><span class="sxs-lookup"><span data-stu-id="adf31-128">Building on top of the base resource grouping mechanism, most cloud platforms provide a system for using templates to deploy your resources to the cloud environment.</span></span> <span data-ttu-id="adf31-129">Vous pouvez utiliser des modèles pour créer une organisation et des conventions de nommage cohérentes lors du déploiement des charges de travail, en appliquant ces aspects de la conception du déploiement et de la gestion de vos ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-129">You can use templates to create consistent organization and naming conventions when deploying workloads, enforcing those aspects of your resource deployment and management design.</span></span>

<span data-ttu-id="adf31-130">Les [modèles Azure Resource Manager](/azure/azure-resource-manager/resource-group-overview#template-deployment) vous permettent de déployer vos ressources de façon répétée et cohérente en utilisant une configuration prédéterminée et une structure de groupes de ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-130">[Azure Resource Manager templates](/azure/azure-resource-manager/resource-group-overview#template-deployment) allow you to repeatedly deploy your resources in a consistent state using a predetermined configuration and resource group structure.</span></span> <span data-ttu-id="adf31-131">Les modèles Resource Manager vous aident à définir un ensemble de standards qui serviront de base à vos déploiements.</span><span class="sxs-lookup"><span data-stu-id="adf31-131">Resource Manager templates help you define a set of standards as a basis for your deployments.</span></span>

## <a name="policy-consistency"></a><span data-ttu-id="adf31-135">Stratégie de cohérence</span><span class="sxs-lookup"><span data-stu-id="adf31-135">Policy consistency</span></span>

<span data-ttu-id="adf31-136">Pour s’assurer que les stratégies de gouvernance sont appliquées lors de la création des ressources, une partie de la conception du regroupement des ressources implique l’utilisation d’une configuration commune lors du déploiement des ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-136">To ensure that governance policies are applied when resources are created, part of resource grouping design involves using a common configuration when deploying resources.</span></span>

<span data-ttu-id="adf31-138">Par exemple, l’une de vos exigences est que toutes les machines virtuelles déployées dans votre abonnement doivent se connecter à un sous-réseau commun géré par votre équipe informatique centrale.</span><span class="sxs-lookup"><span data-stu-id="adf31-138">For example, you may have a requirement that all virtual machines deployed within your subscription connect to a common subnet managed by your central IT team.</span></span> <span data-ttu-id="adf31-139">Vous pouvez créer un modèle standard pour le déploiement des machines virtuelles de charge de travail qui créerait un groupe de ressources distinct pour la charge de travail et y déploierait les machines virtuelles requises.</span><span class="sxs-lookup"><span data-stu-id="adf31-139">You can create a standard template for deploying workload VMs which would create a separate resource group for the workload and deploy the required VMs there.</span></span> <span data-ttu-id="adf31-140">Ce groupe de ressources disposerait d’une règle de stratégie permettant uniquement de joindre les interfaces réseau du groupe de ressources au sous-réseau partagé.</span><span class="sxs-lookup"><span data-stu-id="adf31-140">This resource group would have a policy rule to only allow network interfaces within the resource group to be joined to the shared subnet.</span></span>

## <a name="hierarchical-consistency"></a><span data-ttu-id="adf31-142">Cohérence hiérarchique</span><span class="sxs-lookup"><span data-stu-id="adf31-142">Hierarchical consistency</span></span>

<span data-ttu-id="adf31-143">Au fur et à mesure que la taille de votre investissement cloud augmente, vous aurez peut-être des exigences de gouvernance plus complexes que celles qui peuvent être prises en charge par la hiérarchie Entreprise/Service/Compte /Abonnement de l’accord Entreprise Azure.</span><span class="sxs-lookup"><span data-stu-id="adf31-143">As the size of your cloud estate grows, you may need to support more complicated governance requirements than can be supported using the Azure Enterprise Agreement's Enterprise/Department/Account/Subscription hierarchy.</span></span> <span data-ttu-id="adf31-144">Les groupes de ressources vous permettent de prendre en charge des niveaux hiérarchiques supplémentaires au sein de votre organisation, en appliquant les règles Azure Policy et les contrôles d’accès au niveau d’un groupe de ressources.</span><span class="sxs-lookup"><span data-stu-id="adf31-144">Resource groups allows you to support additional levels of hierarchy within your organization, applying Azure Policy rules and access controls at a resource group level.</span></span>

## <a name="automated-consistency"></a><span data-ttu-id="adf31-147">Cohérence automatisée</span><span class="sxs-lookup"><span data-stu-id="adf31-147">Automated consistency</span></span>
