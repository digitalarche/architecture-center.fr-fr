## <a name="solution"></a><span data-ttu-id="25ddd-126">Solution</span><span class="sxs-lookup"><span data-stu-id="25ddd-126">Solution</span></span>

<span data-ttu-id="25ddd-127">La solution consiste à implémenter une transaction de compensation.</span><span class="sxs-lookup"><span data-stu-id="25ddd-127">The solution is to implement a compensating transaction.</span></span> <span data-ttu-id="25ddd-128">Les étapes d’une transaction de compensation doivent annuler les effets des étapes de l’opération d’origine.</span><span class="sxs-lookup"><span data-stu-id="25ddd-128">The steps in a compensating transaction must undo the effects of the steps in the original operation.</span></span> <span data-ttu-id="25ddd-129">Une transaction de compensation ne peut pas simplement remplacer l’état actuel par l’état dans lequel se trouvait le système au début de l’opération car cette approche pourrait annuler les modifications apportées par d’autres instances simultanées d’une application.</span><span class="sxs-lookup"><span data-stu-id="25ddd-129">A compensating transaction might not be able to simply replace the current state with the state the system was in at the start of the operation because this approach could overwrite changes made by other concurrent instances of an application.</span></span> <span data-ttu-id="25ddd-130">Au lieu de cela, il faut mettre en place un processus intelligent qui prend en compte toute tâche effectuée par les instances simultanées.</span><span class="sxs-lookup"><span data-stu-id="25ddd-130">Instead, it must be an intelligent process that takes into account any work done by concurrent instances.</span></span> <span data-ttu-id="25ddd-131">Ce processus varie généralement en fonction de l’application et de la nature du travail effectué par l’opération d’origine.</span><span class="sxs-lookup"><span data-stu-id="25ddd-131">This process will usually be application specific, driven by the nature of the work performed by the original operation.</span></span>

<span data-ttu-id="25ddd-132">Une approche courante consiste à utiliser un flux de travail pour implémenter une opération cohérente qui nécessite une compensation.</span><span class="sxs-lookup"><span data-stu-id="25ddd-132">A common approach is to use a workflow to implement an eventually consistent operation that requires compensation.</span></span> <span data-ttu-id="25ddd-133">Au cours de l’exécution de l’opération d’origine, le système enregistre des informations sur chaque étape et identifie la manière dont le travail effectué par l’étape en cours peut être annulé.</span><span class="sxs-lookup"><span data-stu-id="25ddd-133">As the original operation proceeds, the system records information about each step and how the work performed by that step can be undone.</span></span> <span data-ttu-id="25ddd-134">Si l’opération échoue à un moment donné, le flux de travail effectue un retour en arrière sur les étapes effectuées et annule chacune de ces étapes.</span><span class="sxs-lookup"><span data-stu-id="25ddd-134">If the operation fails at any point, the workflow rewinds back through the steps it's completed and performs the work that reverses each step.</span></span> <span data-ttu-id="25ddd-135">Notez qu’une transaction de compensation n’annule pas forcément les étapes dans l’ordre où elles ont été effectuées dans l’opération d’origine et qu’il est possible d’effectuer certaines des étapes d’annulation en parallèle.</span><span class="sxs-lookup"><span data-stu-id="25ddd-135">Note that a compensating transaction might not have to undo the work in the exact reverse order of the original operation, and it might be possible to perform some of the undo steps in parallel.</span></span>

> <span data-ttu-id="25ddd-136">Cette approche est similaire à la stratégie Sagas présentée sur [le blog de Clemens Vasters](https://vasters.com/clemensv/2012/09/01/Sagas.aspx).</span><span class="sxs-lookup"><span data-stu-id="25ddd-136">This approach is similar to the Sagas strategy discussed in [Clemens Vasters’ blog](https://vasters.com/clemensv/2012/09/01/Sagas.aspx).</span></span>

<span data-ttu-id="25ddd-137">Une transaction de compensation est également une opération de cohérence éventuelle qui peut échouer.</span><span class="sxs-lookup"><span data-stu-id="25ddd-137">A compensating transaction is also an eventually consistent operation and it could also fail.</span></span> <span data-ttu-id="25ddd-138">Le système doit être en mesure de relancer la transaction de compensation à partir du point de défaillance et de poursuivre son exécution.</span><span class="sxs-lookup"><span data-stu-id="25ddd-138">The system should be able to resume the compensating transaction at the point of failure and continue.</span></span> <span data-ttu-id="25ddd-139">Il peut être nécessaire de répéter une étape qui a échoué. Les étapes d’une transaction de compensation doivent donc être définies en tant que commandes idempotentes.</span><span class="sxs-lookup"><span data-stu-id="25ddd-139">It might be necessary to repeat a step that's failed, so the steps in a compensating transaction should be defined as idempotent commands.</span></span> <span data-ttu-id="25ddd-140">Pour plus d’informations, consultez l’article [Idempotency Patterns](https://blog.jonathanoliver.com/idempotency-patterns/) sur le blog de Jonathan Oliver.</span><span class="sxs-lookup"><span data-stu-id="25ddd-140">For more information, see [Idempotency Patterns](https://blog.jonathanoliver.com/idempotency-patterns/) on Jonathan Oliver’s blog.</span></span>

<span data-ttu-id="25ddd-141">Dans certains cas, il n’est pas possible de reprendre l’opération à partir d’une étape qui a échoué, sauf via une intervention manuelle.</span><span class="sxs-lookup"><span data-stu-id="25ddd-141">In some cases it might not be possible to recover from a step that has failed except through manual intervention.</span></span> <span data-ttu-id="25ddd-142">Dans ces situations, le système doit générer une alerte et fournir autant d’informations que possible sur la raison de l’échec.</span><span class="sxs-lookup"><span data-stu-id="25ddd-142">In these situations the system should raise an alert and provide as much information as possible about the reason for the failure.</span></span>

## <a name="issues-and-considerations"></a><span data-ttu-id="25ddd-143">Problèmes et considérations</span><span class="sxs-lookup"><span data-stu-id="25ddd-143">Issues and considerations</span></span>

<span data-ttu-id="25ddd-148">-Une logique de compensation peut difficilement être généralisée.</span><span class="sxs-lookup"><span data-stu-id="25ddd-148">-Compensation logic isn't easily generalized.</span></span> <span data-ttu-id="25ddd-149">La transaction de compensation varie en fonction de chaque application.</span><span class="sxs-lookup"><span data-stu-id="25ddd-149">A compensating transaction is application specific.</span></span> <span data-ttu-id="25ddd-150">Elle s’appuie sur les informations dont dispose l’application pour pouvoir annuler les effets de chaque étape d’une opération ayant échoué.</span><span class="sxs-lookup"><span data-stu-id="25ddd-150">It relies on the application having sufficient information to be able to undo the effects of each step in a failed operation.</span></span>
